# Шпаргалка 20: Публикация Allure отчетов в GitHub Pages

Запуск тестов в CI/CD — это половина дела. Менеджеры, разработчики и вы сами должны видеть удобные графики прохождения тестов, скриншоты и логи упавших сценариев. Для этого отчет нужно публиковать (деплоить).

## Как это работает?
1. Наши автотесты генерируют сырые данные в папке `allure-results`.
2. С помощью специального инструмента (`allure-report-action`) сырые данные превращаются в красивый HTML-сайт.
3. С помощью второго экшена (`actions-gh-pages`) мы "пушим" этот сайт в специальную ветку `gh-pages` нашего репозитория.
4. Сервис GitHub Pages делает эту ветку доступной по публичной веб-ссылке.

## Важные особенности YAML-настройки
Чтобы экшен смог сделать коммит в ветку `gh-pages`, у него должны быть на это права (Permissions). Поэтому мы добавляем в начало `playwright.yml`:
```yaml
permissions:
  contents: write
  pages: write
```

Также для шагов создания отчета мы всегда используем инструкцию `if: always()`. 
Зачем? По умолчанию GitHub Actions прерывает работу, если предыдущий шаг упал с ошибкой. Если автотесты упадут, мы всё равно **обязаны** сгенерировать отчет, чтобы посмотреть скриншоты и логи и понять, почему произошла ошибка!

## Настройка GitHub (после первого запуска)
1. Перейдите в вашем репозитории в **Settings** -> **Pages**.
2. В блоке **Build and deployment** выберите **Deploy from a branch**.
3. В выпадающем списке выберите ветку `gh-pages`, папку `/(root)` и нажмите **Save**.
4. Через минуту вверху страницы появится ссылка вида: `https://вашинник.github.io/Cemini_automation_project/`. Это и есть ваш постоянный портал с отчетами!
