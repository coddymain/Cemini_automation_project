# Шпаргалка 12: Интеграция Allure и Playwright (Хуки Pytest)

В Python экосистеме интеграция отчетов (Allure) и инструментов автоматизации (Playwright) требует небольшой "склейки".

## 1. В чем проблема?

По умолчанию:
*   `pytest-playwright` делает скриншоты при падении и сохраняет их в папку `test-results/`.
*   `allure-pytest` генерирует отчет на основе данных из своей папки `allure-results/`.
Они изолированы друг от друга. Отчет Allure не знает, что в соседней папке лежит нужный скриншот.

## 2. Решение: Механизм Хуков (Hooks) в Pytest

Pytest предоставляет мощный механизм — **Хуки (Hooks)**. Хук — это функция, которая позволяет "вклиниться" в стандартный процесс выполнения тестов. 

Мы используем хук `pytest_runtest_makereport`, который вызывается Pytest'ом для генерации отчета о каждой фазе теста.

### Жизненный цикл теста:
Любой тест в Pytest состоит из 3 фаз:
1.  **setup**: Выполнение фикстур, открытие браузера.
2.  **call**: Выполнение самого тела теста (строк внутри `def test_...()`).
3.  **teardown**: Очистка, закрытие браузера.

Нам нужно перехватить падение именно на этапе **call**.

## 3. Разбор кода хука (из conftest.py)

```python
@pytest.hookimpl(tryfirst=True, hookwrapper=True)
def pytest_runtest_makereport(item, call):
    # 1. Запускаем оригинальную логику Pytest и ждем результат
    outcome = yield
    report = outcome.get_result()

    # 2. Фильтруем: нас интересует только этап 'call' (само выполнение теста), и только если была ошибка (failed)
    if report.when == "call" and report.failed:
        
        # 3. item - это абстракция текущего теста. 
        # item.funcargs содержит все фикстуры, переданные в тест. Мы достаем фикстуру 'page' (наш браузер).
        page = item.funcargs.get('page')
        
        if page:
            try:
                # 4. Просим Playwright сфотографировать текущее состояние браузера.
                # Скриншот создается в оперативной памяти (в виде набора байтов), без сохранения на диск!
                screenshot = page.screenshot(type='png', full_page=True)
                
                # 5. Передаем байты скриншота в Allure API.
                # slugify(item.name) превращает "test my login" в "test-my-login", чтобы имя файла было безопасным.
                allure.attach(
                    screenshot,
                    name=f"Screenshot_{slugify(item.name)}",
                    attachment_type=allure.attachment_type.PNG
                )
            except Exception as e:
                print(f"Не удалось сделать скриншот для Allure: {e}")
```

## 4. Почему мы не прикрепляем Видео таким же образом?

Многие новички пытаются сразу прикрепить к Allure и видеофайл `video.webm`. Однако в Playwright (через pytest) видеофайл окончательно записывается на диск **только в фазе teardown** (когда закрывается страница/контекст браузера).

На этапе падения (`call`) файла с видео еще физически не существует в файловой системе.

**Best Practice в индустрии:**
*   Скриншоты прикрепляются напрямую в Allure-отчет (для быстрого визуального понимания проблемы).
*   Для глубокого анализа (замена видео) используется файл **Trace** (`trace.zip`), который можно посмотреть через команду `playwright show-trace`. Видео занимает много места и дает меньше информации, чем интерактивный Трейс.
