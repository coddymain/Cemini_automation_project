# Шпаргалка: Git, GitHub и CI/CD

## 1. Базовые команды Git (Локальная работа)

- `git init` 
  *Зачем:* Инициализирует пустой локальный репозиторий (создает скрытую папку `.git`). Делается один раз для проекта.

- `git status`
  *Зачем:* Показывает текущее состояние файлов: какие изменены, какие новые (untracked), какие готовы к коммиту (staged).

- `git add .`
  *Зачем:* Добавляет все измененные и новые файлы (кроме тех, что в `.gitignore`) в индекс (staging area), подготавливая их к сохранению. Точка `.` означает "всё в текущей папке".

- `git commit -m "Текст сообщения"`
  *Зачем:* Создает "снимок" (коммит) подготовленных файлов. Флаг `-m` позволяет сразу написать сообщение коммита. Сообщение должно быть понятным (что именно изменено).

## 2. Работа с GitHub (Удаленный репозиторий)

- `git remote add origin <ССЫЛКА_ИЗ_GITHUB>`
  *Зачем:* Связывает локальный репозиторий с удаленным (на GitHub). `origin` — это стандартное имя-псевдоним для главной удаленной ссылки.

- `git branch -M main`
  *Зачем:* Принудительно (флаг `-M`) переименовывает текущую локальную ветку в `main` (современный стандарт вместо `master`).

- `git push -u origin main`
  *Зачем:* Отправляет локальные коммиты на сервер (в `origin`) в ветку `main`. Флаг `-u` запоминает эту связь, чтобы в будущем можно было писать просто `git push`.

## 3. GitHub Actions (CI/CD)

- **Где лежат настройки:** `.github/workflows/*.yml` (например, `tests.yml`).
- **Основные блоки файла .yml:**
  - `on:` — Триггеры (когда запускать сценарий, например, при `push` в ветку).
  - `jobs:` — Задачи (что именно делать).
  - `runs-on: ubuntu-latest` — Выбор операционной системы сервера, на котором будет выполняться работа.
  - `steps:` — Пошаговые инструкции (скачать код, установить программы, запустить скрипты).
  - `uses:` — Вызов готового чужого "экшена" (например, `actions/checkout` для скачивания кода).
  - `run:` — Выполнение консольной команды Linux (например, `python --version`).
