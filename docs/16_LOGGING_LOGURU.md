# Шпаргалка 16: Индустриальное логирование с Loguru

В автоматизации тестирования стандартные команды `print()` использовать нельзя. При запуске сотен тестов в облаке вам нужно четко понимать: когда произошел клик, в каком файле, и был ли это штатный шаг (INFO) или ошибка (ERROR). 
В этой шпаргалке описана **полная последовательность действий** по внедрению мощного логирования с нуля.

## Шаг 1. Подготовка и Установка
Библиотека **Loguru** требует строгой совместимости версий Python. 
1. Убедитесь, что в вашем файле `pyproject.toml` в блоке `[project]` версия Python ограничена сверху (должно быть `requires-python = ">=3.12,<4.0"`). Если там просто `>=3.12`, Poetry выдаст ошибку при установке.
2. Выполните в терминале в корне проекта команду для установки:
```bash
poetry add loguru
```

## 2. Активация логгера
Чтобы конфигурация применилась ко всему проекту, логгер инициализируется на самом раннем этапе. 
В нашем проекте мы делаем это в начале файла `conftest.py`, вызывая функцию `setup_logger()`.
## Шаг 2. Создание конфигурации (Ядро логирования)
Создайте файл `core/logger.py` и добавьте в него базовую конфигурацию. Этот файл будет централизованно управлять форматом логов во всем проекте.

## 3. Как использовать в Page Objects и Тестах
Теперь в любом файле страниц (например, в `pages/login_page.py`) или тестов мы импортируем объект `logger` напрямую из библиотеки (важно: не из нашего `core`, а из `loguru` — конфигурация подтянется автоматически).

**Пример:**
```python
import sys
from loguru import logger

class LoginPage(BasePage):
    def login(self, username, password):
        logger.info(f"Начинаем авторизацию под пользователем: {username}")
        self._username_input.fill(username)
        self._password_input.fill(password)
        self._login_button.click()
        logger.success("Кнопка логина нажата")
```

### Основные уровни логирования:
*   `logger.debug("...")` — Технические детали (например, "Ищем локатор X"). Полезно для отладки.
*   `logger.info("...")` — Бизнес-шаги (открываем страницу, логинимся, добавляем в корзину).
*   `logger.success("...")` — Успешное завершение сложной операции.
*   `logger.warning("...")` — Предупреждение (например, "Элемент не найден с первого раза, пробуем еще").
*   `logger.error("...")` — Критическая ошибка, из-за которой тест скорее всего упадет.

## 4. Разбор конфигурации (`core/logger.py`)
Мы добавили этот файл на этапе подготовки проекта к Enterprise-стандартам (когда тесты стали запускаться в CI/CD). Он нужен для того, чтобы централизованно управлять форматом логов во всем проекте и отказаться от неинформативных `print()`.

```python
def setup_logger() -> None:
    """
    Настройка кастомного логгера для всего проекта.
    """
    # Удаляем стандартный обработчик, чтобы логи не дублировались
    logger.remove()

    # 1. Обработчик для консоли (stdout) - цветной, красивый вывод
    logger.add(
        sys.stdout,
        format="<green>{time:YYYY-MM-DD HH:mm:ss}</green> | <level>{level: <8}</level> | <cyan>{name}</cyan>:<cyan>{function}</cyan>:<cyan>{line}</cyan> - <level>{message}</level>",
        level="INFO",
        colorize=True
    )

    # 2. Обработчик для записи в файл (сохраняем полную историю тестов)
    logger.add(
        "logs/test_run.log",
        format="{time:YYYY-MM-DD HH:mm:ss} | {level: <8} | {name}:{function}:{line} - {message}",
        level="DEBUG",
        rotation="5 MB",    # Создавать новый файл, когда старый достигнет 5 Мегабайт
        compression="zip"   # Старые логи автоматически архивируются
    )
```
### Почему именно такие настройки?
*   **Форматирование:** Задана четкая структура — время, уровень, файл, функция, строка кода. Без этого при падении теста в CI тяжело понять, где именно произошла ошибка.
*   **Умная ротация (`rotation="5 MB"`):** Защищает сервер и локальный жесткий диск от переполнения гигантскими текстовыми файлами.
*   **Разделение уровней:** В консоль идет только важное (`INFO`), в файл пишется всё вплоть до мелочей (`DEBUG`). Это экономит время при чтении логов в терминале, но сохраняет "черный ящик" на случай сложных багов.

## 5. Практическая дорожная карта: Как запустить логирование в проекте

Когда "движок" логирования (`core/logger.py`) создан, проект о нем еще ничего не знает. Чтобы всё заработало, нужно последовательно внедрить его в ключевые файлы.

### Шаг 1: Запуск "движка" логирования (`conftest.py`)
*   **Для чего это:** Pytest должен применить наши правила логирования (цветной текст, сохранение логов в файл) в самую первую секунду работы, еще до запуска тестов.
*   **Куда вставляем:** В самый верх файла `conftest.py`, сразу после импортов. 
*   **Дополнительно:** В хуке `pytest_runtest_makereport` мы добавляем `allure.attach.file(...)`, чтобы лог-файл автоматически прикреплялся к отчету при падении теста.
```python
from core.logger import setup_logger

# Инициализируем настройки логгера для всего проекта при старте тестов
setup_logger()
```

### Шаг 2: Расстановка "датчиков" на странице входа (`login_page.py`)
*   **Для чего это:** Страницы (Page Objects) должны отчитываться о том, что они делают. Вместо комментариев `# вводим логин`, мы заставляем код "говорить" в консоль.
*   **Откуда берем:** Напрямую из библиотеки `loguru` (`from loguru import logger`).
*   **Куда вставляем:** В методы действий (там, где происходят клики и ввод текста).
```python
from loguru import logger

# Внутри метода login:
logger.info(f"Выполняем вход под пользователем: {username}")
# ... действия ...
logger.success("Кнопка входа успешно нажата")
```

### Шаг 3: Расстановка "датчиков" на странице товаров (`inventory_page.py`)
*   **Для чего это:** Чтобы видеть в консоли шаги добавления товаров в корзину и переход дальше.
```python
from loguru import logger

# Внутри метода add_backpack_to_cart:
logger.info("Добавляем рюкзак 'Sauce Labs Backpack' в корзину")
# ... клик ...
logger.success("Рюкзак добавлен")
```

### Шаг 4: Запуск и проверка
Запустите автотесты в терминале: 
```bash
poetry run pytest -v
```
**Результат:** Вы увидите цветные сообщения шагов в консоли, а слева в дереве проекта появится папка `logs` с файлом `test_run.log`.
