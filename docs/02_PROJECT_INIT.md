# Шпаргалка 2: Инициализация проекта и Архитектура папок

Любой Enterprise-проект начинается с правильной структуры и изоляции зависимостей.

## 1. Структура папок (Enterprise Directory Structure)

Мы создали следующие папки и файлы:

*   📂 `tests/` — Здесь будут лежать сами файлы с тестами (например, `test_login.py`).
*   📂 `pages/` — Здесь мы будем описывать наши страницы (Page Objects). Одна страница сайта = один файл класса.
*   📂 `data/` — Хранилище тестовых данных (модели Pydantic, классы с товарами, тестовые пользователи).
*   📂 `core/` — "Сердце" инфраструктуры. Настройки проекта, чтение конфигов, инициализация базовых классов.
*   📄 `.env` — Скрытый файл для паролей и ссылок. **Важно:** Этот файл мы сразу добавляем в `.gitignore`.
*   📄 `.gitignore` — Файл, в котором мы перечисляем, что **не должно** попасть в систему контроля версий (Git). Мы не пушим в Git логины, виртуальные окружения и временные файлы.
*   📄 `pytest.ini` — Главный конфигурационный файл для Pytest. Здесь мы будем настраивать параметры запуска тестов по умолчанию (например, параллельность или создание отчетов).

## 2. Инициализация Poetry (Современная альтернатива requirements.txt)

Вместо `pip install ...` и `requirements.txt` мы используем **Poetry**.

**Основные команды Poetry (на будущее):**
*   `poetry init` — Инициализирует проект (создает файл `pyproject.toml`, где будет храниться список всех нужных библиотек).
*   `poetry add <название>` — Установить библиотеку (аналог `pip install`, но с умным сохранением версий). Пример: `poetry add pytest`.
*   `poetry add <название> --group dev` — Установить библиотеку только для разработки (тестирования). На продакшен она не пойдет.
*   `poetry run pytest` — Запустить команду `pytest` внутри виртуального изолированного окружения этого проекта.

### Зачем нам `pyproject.toml`?
Этот файл пришел на смену `setup.py` и `requirements.txt`. Это единый современный стандарт в Python для описания проекта (имя, версия, зависимости, настройки линтеров и фреймворков).

В следующем шаге мы инициализируем Poetry и установим наши главные инструменты: **pytest**, **playwright** и **allure**.
