# Шпаргалка 3: Современное управление зависимостями (Poetry)

В Enterprise-разработке мы больше не используем стандартный `pip install ...` и файл `requirements.txt`. Вместо этого мы используем **Poetry**.

## Почему Poetry, а не pip?
1.  **Жесткая фиксация (Locking):** Poetry создает файл `poetry.lock`. Он запоминает не только версии библиотек, которые вы запросили, но и версии **всех их внутренних зависимостей** до самого низа. Это гарантирует, что проект запустится одинаково на любом компьютере (или сервере CI/CD) даже спустя год.
2.  **Изоляция (Virtual Environments):** Poetry сам создает и управляет виртуальным окружением (`.venv`). Вам не нужно делать это вручную (`python -m venv venv`).
3.  **Разделение зависимостей:** Можно отдельно ставить библиотеки для продакшена и отдельно для тестов/разработки.

## Пошаговая инициализация, которую мы сделали:

### Шаг 1. Установка самого Poetry
Сначала мы установили Poetry на ваш компьютер (macOS):
```bash
pip install poetry
```
*Эта команда устанавливает менеджер пакетов глобально в вашу систему.*

### Шаг 2. Инициализация проекта
Затем, находясь в папке проекта (`Cemini_automation_project`), мы выполнили команду:
```bash
poetry init -n --name cemini-automation-project --description "Enterprise QA Automation" --author "Dmitrij Erastov"
```
*   `poetry init` — Команда для начала работы с проектом. Она создает главный файл конфигурации `pyproject.toml`.
*   `-n` (или `--no-interaction`) — Флаг, который говорит программе: "не задавай мне вопросов в терминале, используй стандартные значения или те, что я передал во флагах".
*   `--name`, `--description`, `--author` — Флаги, передающие информацию о проекте.

После этой команды в корне проекта появился файл **`pyproject.toml`**. Это "паспорт" нашего проекта.

### Шаг 3. Настройка окружения
Мы сказали Poetry создавать папку с виртуальным окружением прямо внутри нашего проекта (чтобы VS Code ее сразу увидел):
```bash
poetry config --local virtualenvs.in-project true
```
*   `poetry config` — Команда настройки.
*   `--local` — Применить настройку только для этой папки (создастся файл `poetry.toml` с этой настройкой).
*   `virtualenvs.in-project true` — Само правило: "храни окружение в папке `.venv` внутри текущего каталога".

### Шаг 4. Установка библиотек (Мы сейчас на этом шаге)
Мы запустили установку всех необходимых библиотек:
```bash
poetry add pytest pytest-playwright allure-pytest pydantic pydantic-settings
```
*   `poetry add` — Команда добавления и скачивания библиотеки (аналог `pip install`).
*   `pytest` — Главный фреймворк для написания и запуска тестов.
*   `pytest-playwright` — Интеграция браузерного движка Playwright в Pytest (для UI-тестов).
*   `allure-pytest` — Интеграция с системой отчетов Allure.
*   `pydantic` и `pydantic-settings` — Библиотеки для строгой проверки данных (типизации) и удобной работы с файлом `.env`.

**Важно:** Установка может занимать до 5-10 минут, так как скачивается `playwright` (тяжелый движок для работы с браузерами). В этот момент Poetry создает папку `.venv` и скачивает туда весь код библиотек.
