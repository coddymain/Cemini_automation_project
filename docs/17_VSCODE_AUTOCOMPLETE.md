# Шпаргалка 17: Почему не работает автодополнение (Autocomplete) и зачем нужны импорты?

Частая проблема при написании автотестов: вы пишете название вашей страницы (например, `checkout_overview_page`), ставите точку `.`, но редактор кода (VS Code) не предлагает список методов (например, `finish_checkout`), и вам приходится печатать всё вручную по памяти.

## 1. В чем причина?
Редактор кода "глупый" — он не умеет читать ваши мысли и не знает, как работает магия фикстур Pytest из `conftest.py`. 
Когда вы пишете в аргументах теста `def test_e2e(checkout_overview_page):`, VS Code видит просто какое-то слово. Он не знает, что скрывается за этим словом (какой это класс, в каком файле он лежит и какие у него методы).

## 2. Решение: Type Hints (Аннотации типов)
Чтобы VS Code "прозрел" и начал помогать вам, ему нужно явно указать тип переменной. В Python это делается через двоеточие: `переменная: Тип`.
Но чтобы редактор распознал этот `Тип`, **сам класс страницы нужно обязательно импортировать** в верхней части файла.

### Правильный шаблон:

```python
# 1. ИМПОРТ: Говорим редактору, где лежит инструкция к странице
from pages.checkout_overview_page import CheckoutOverviewPage

# 2. ТИПИЗАЦИЯ (Type Hint): Говорим, что аргумент относится к этому классу
def test_full_checkout_flow(checkout_overview_page: CheckoutOverviewPage):
    
    # 3. МАГИЯ АВТОДОПОЛНЕНИЯ: Ставим точку, и редактор сам предлагает методы!
    checkout_overview_page.finish_checkout()
```

## 3. Важное правило архитектуры (Разделение обязанностей)
Может возникнуть вопрос: *"Если мы импортируем `CheckoutOverviewPage` в файл теста, почему мы сами не создаем его, а всё равно используем фикстуры из `conftest.py`?"*

*   **Импорт класса в файле теста** нужен **ТОЛЬКО** для редактора кода (VS Code), чтобы он показывал вам подсказки при написании кода.
*   **Фикстура в аргументах** нужна для **Pytest**, чтобы во время запуска он сам пошел в `conftest.py`, создал там реальный объект страницы, открыл в нем браузер и принес готовый результат в тест.

Мы не пишем внутри теста `page_object = CheckoutOverviewPage(page)` — это нарушение паттерна внедрения зависимостей (Dependency Injection). Мы разделяем ответственность: Pytest собирает страницы, а VS Code через импорты просто помогает нам писать код без опечаток.

## Чек-лист: Что делать, если пропали подсказки после точки?
1. Проверьте, есть ли сверху импорт класса: `from pages... import НазваниеКласса`
2. Проверьте аргументы функции теста: написано ли там `ваша_фикстура: НазваниеКласса`
3. Проверьте, совпадает ли написание `НазваниеКласса` там и там (с учетом заглавных букв).
