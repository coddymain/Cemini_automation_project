# Шпаргалка 5: Настройка ядра Pytest (conftest.py и фикстуры)

В этом разделе мы настроили "сердце" наших автотестов — конфигурацию Pytest и интеграцию с Playwright.

## 1. Что такое conftest.py?
`conftest.py` — это специальный файл в экосистеме Pytest. 
Его главная особенность: **все фикстуры (fixtures), объявленные в нем, автоматически доступны всем тестам в проекте.** Их не нужно импортировать вручную.

## 2. Использование плагина pytest-playwright
Вместо того чтобы писать "велосипед" — код для запуска браузера, создания контекста и открытия страницы с нуля, мы используем готовый мощный плагин `pytest-playwright`.
Он "из коробки" предоставляет нам умные фикстуры:
*   `browser` — инстанс браузера.
*   `context` — изолированное окружение (кэш, куки).
*   `page` — чистая вкладка браузера для каждого теста.

## 3. Настройка контекста (browser_context_args)
Вместо переписывания базовых фикстур, мы **дополняем** их. В нашем `conftest.py` мы переопределили фикстуру `browser_context_args`, чтобы задать глобальные настройки для каждой открываемой страницы:

```python
import pytest
from typing import Dict, Any
from core.settings import config

@pytest.fixture(scope="session")
def browser_context_args(browser_context_args: Dict[str, Any]) -> Dict[str, Any]:
    return {
        **browser_context_args,
        "base_url": config.BASE_URL,
        "viewport": {"width": 1920, "height": 1080},
        "ignore_https_errors": True,
    }
```

### Разбор кода:
*   `scope="session"`: Фикстура отрабатывает один раз за всю сессию тестирования, применяя настройки ко всем тестам.
*   `**browser_context_args`: Распаковываем стандартные настройки плагина, чтобы не потерять их.
*   `base_url`: Позволяет в тестах писать `page.goto("/")` вместо полного адреса. Берем из нашего `.env` через Pydantic.
*   `viewport`: Задаем строгое разрешение экрана для избежания "плавающих" багов верстки.

## 4. Enterprise-стандарты (Type Hints и Docstrings)
Важный урок из этого этапа: в современных проектах уровня Middle/Senior обязательна строгая типизация. 
Мы используем **Type Hints** (`Dict[str, Any]`), чтобы IDE (VS Code) и статические анализаторы (mypy) понимали, какие типы данных входят в функцию и какие из нее выходят. Также обязательны **Docstrings** (комментарии в тройных кавычках) для объяснения *цели* кода для других разработчиков.
