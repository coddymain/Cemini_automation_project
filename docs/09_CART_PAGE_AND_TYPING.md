# Шпаргалка 9: Создание новых страниц, строгая типизация (Type Hinting) и методы действий

В этом этапе мы закрепили знания по созданию страниц (Page Objects) на примере корзины (`CartPage`), а также углубились в стандарты Enterprise-кода: строгую типизацию и разделение локаторов от действий.

## 1. Строгая типизация (Type Hinting) в Python

Когда мы пишем код, очень важно, чтобы и мы, и наш редактор (VS Code) понимали, с какими типами данных мы работаем. Это защищает от глупых ошибок (например, попытки вызвать метод `.click()` у строки текста) и включает автодополнение.

```python
from playwright.sync_api import Locator

class CartPage(BasePage):
    @property
    def checkout_button(self) -> Locator:
        return self.page.locator('[data-test="checkout"]')
```

*   **`from playwright.sync_api import Locator`**: Мы явно импортируем класс `Locator`. Это тип данных, который Playwright использует для представления элементов на странице.
*   **`-> Locator`**: Это **Type Hint (Подсказка типа)**. Она говорит: "Этот метод/свойство всегда будет возвращать объект типа Locator, и ничего другого".
*   **Польза**: Когда в тесте мы напишем `cart_page.checkout_button.`, редактор сразу предложит нам `.click()`, `.fill()`, `.is_visible()` и другие методы, доступные именно для локаторов Playwright.

## 2. Разделение: Локаторы и Действия

В хорошем Page Object классе мы визуально и логически разделяем две вещи: **Что мы ищем** (Локаторы) и **Что мы с этим делаем** (Действия).

### Локаторы (Свойства)
Они только находят элемент, но ничего с ним не делают. Мы помечаем их декоратором `@property`, чтобы к ним можно было обращаться как к атрибутам (`self.checkout_button`), а не как к функциям (`self.checkout_button()`).

```python
    """Локаторы"""
    @property
    def checkout_button(self) -> Locator:
        return self.page.locator('[data-test="checkout"]')
```

### Действия (Методы)
Это функции, которые описывают бизнес-логику (пользовательский сценарий). Они используют наши локаторы и выполняют над ними действия Playwright.
Они всегда имеют тип возвращаемого значения `-> None`, если просто выполняют действие и ничего не возвращают обратно в тест.

```python
    """Действия"""
    def click_checkout(self) -> None:
        """Кликает по кнопке Checkout для перехода к оформлению заказа."""
        self.checkout_button.click()
```

## 3. Регистрация страницы в conftest.py

Создать класс страницы — это только полдела. Чтобы наши тесты могли легко получать к ней доступ через магию фикстур, страницу нужно "зарегистрировать" (создать для нее поставщика) в центральном файле `conftest.py`.

```python
from pages.cart_page import CartPage

@pytest.fixture
def cart_page(page: Page) -> CartPage:
    """Фикстура для инициализации CartPage."""
    return CartPage(page)
```

Теперь любой тест может просто попросить `cart_page` в своих аргументах, и Pytest сам создаст свежий экземпляр страницы и передаст его в тест!
