# Итоговый план проекта: Enterprise QA Automation (Python)

## 🎯 1. Цель и Стек (Goal and Stack)
*   **Стек:** Python, Playwright, Pytest. Только современные подходы (актуальные на 2026 год).
*   **Среда разработки:** macOS, редактор VS Code.
*   **Менеджер пакетов (Package Manager):** Poetry (вместо устаревшего requirements.txt) для жесткой фиксации версий и автоматического управления окружением.
*   **Итоговая цель:** Уровень QA Automation Engineer Middle. Умение с нуля строить масштабируемую архитектуру, писать чистый код и понимать каждую строчку.

## 🏗 2. Архитектура и Подходы (Architecture and Approaches)
*   **Структура:** Enterprise (корпоративная) архитектура. Проект должен легко масштабироваться на тысячи тестов.
*   **Управление данными (Data Management):** 
    *   Файлы `.env` для хранения секретов (логины, пароли, базовые URL).
    *   `Pydantic` или `Data Classes` для строгой типизации тестовых данных (например, информации о товарах).
*   **Паттерны (Patterns):** Современная реализация Page Object Model (POM), активное использование фикстур (fixtures) Pytest для внедрения зависимостей (Dependency Injection) и управления состоянием.

## 🎯 3. Что тестируем (Target)
*   **Приложение:** Swag Labs (https://www.saucedemo.com/).
*   **Фокус:** UI (пользовательский интерфейс).
*   **Главный сценарий:** E2E (End-to-End). Полный путь пользователя от авторизации до успешного оформления заказа.

## 🚀 4. Выполнение и Отчетность (Execution and Reporting)
*   **Запуск (Execution):** 
    1.  Последовательный (sequentially) — для детального разбора и отладки.
    2.  Параллельный (parallel execution) через `pytest-xdist` — для скорости (настроим позже).
*   **Отчеты (Reporting):** Интеграция с Allure Framework. Обязательное автоматическое прикрепление скриншотов (screenshots) и видеозаписей (videos) прохождения тестов.

## 📚 5. Формат обучения (Learning Format)
*   **Поэтапно:** Создаем файлы один за другим, без генерации всего проекта сразу.
*   **Шпаргалки (Cheat Sheets):** Весь процесс логируется в папке `docs/` в виде Markdown-файлов. Это ваша база знаний.
*   **Детальный разбор:** Каждая строчка кода комментируется: зачем нужна, откуда берется, куда идет.
*   **Без "воды":** Только суть, с примерами. Подсвечиваем старые подходы (как делать не надо) и современные стандарты.
