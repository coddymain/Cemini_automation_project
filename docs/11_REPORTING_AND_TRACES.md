# Шпаргалка 11: Отчетность, Скриншоты, Видео и Трейсы (Allure + Playwright)

В этом этапе мы настроили автоматическую генерацию артефактов при падении тестов и подготовились к работе с Allure-отчетами.

## 1. Зачем это нужно?

Когда тесты запускаются в CI/CD (например, на GitHub Actions или GitLab CI) без графического интерфейса (headless mode), мы не можем видеть экран. Если тест упадет, текст ошибки в консоли не всегда дает полное понимание проблемы.

Нам нужны **артефакты**:
1. **Скриншот (Screenshot):** Картинка того, что было на экране в момент ошибки.
2. **Видео (Video):** Запись всего прохождения теста до момента падения.
3. **Трейс (Trace):** Полный слепок состояния Playwright. Это как "машина времени", где можно посмотреть DOM, сетевые запросы, консоль браузера на каждом шаге теста.
4. **HTML-отчет (Allure Report):** Красивая веб-страница с историей прогонов, графиками и прикрепленными артефактами.

## 2. Настройка `pytest.ini`

Мы настроили эти функции централизованно через файл `pytest.ini`.

```ini
[pytest]
addopts = 
    -v
    --alluredir=allure-results
    --screenshot=only-on-failure
    --video=retain-on-failure
    --tracing=retain-on-failure

markers =
    e2e: End-to-end тесты
    ui: UI тесты
```

### Разбор флагов:

*   `addopts`: Позволяет задать аргументы командной строки, которые `pytest` будет применять автоматически при каждом запуске.
*   `-v`: Verbose режим (более подробный вывод в консоли).
*   `--alluredir=allure-results`: Указывает плагину `allure-pytest` собирать "сырые" JSON-данные о результатах тестов и складывать их в папку `allure-results/`.
*   `--screenshot=only-on-failure`: Playwright сделает скриншот браузера **только** если тест завершится с ошибкой.
*   `--video=retain-on-failure`: Playwright записывает видео всегда, но сохраняет его на диск **только** при падении. Это экономит место.
*   `--tracing=retain-on-failure`: Сохраняет полный trace-файл (в формате `.zip`) при падении.

## 3. Что происходит при падении теста?

Когда мы намеренно сломали тест и запустили его, Playwright и Pytest сгенерировали две важные папки:

1.  **`test-results/` (Создает Playwright):**
    Здесь лежат артефакты самого браузера. В подпапке (например, `tests-test-e2e-py-test-full-checkout-flow-chromium`) мы найдем:
    *   `test-failed-1.png` — скриншот ошибки.
    *   `video.webm` — видео прохождения.
    *   `trace.zip` — файл трейса.

2.  **`allure-results/` (Создает Allure):**
    Здесь лежат JSON-файлы с метаданными о тесте: шаги, время выполнения, статусы, ссылки на артефакты. Это "сырые" данные, которые пока нельзя прочитать глазами.

## 4. Как посмотреть Trace Playwright?

Чтобы открыть файл `trace.zip`, нужно использовать специальный инструмент Playwright:

```bash
poetry run playwright show-trace test-results/<имя-папки>/trace.zip
```

Он откроет локальное веб-приложение, где можно буквально "отмотать" тест назад.

## 5. Как сгенерировать Allure-отчет?

Чтобы превратить сырые JSON-данные из папки `allure-results/` в красивый HTML-отчет, нужно использовать CLI-утилиту Allure:

```bash
allure serve allure-results
```

Эта команда поднимет локальный сервер и откроет отчет в браузере.
