# 🚀 Enterprise QA Automation Project (Swag Labs)

## 📖 О проекте
Данный проект представляет собой современную, масштабируемую архитектуру автоматизированного тестирования на базе Python. 
Проект написан с соблюдением лучших практик индустрии (Enterprise level) для UI и гибридного (API) тестирования.

**Тестируемое приложение:** [Swag Labs](https://www.saucedemo.com/) (Учебный e-commerce портал).

## 🛠 Технологический стек
* **Язык программирования:** Python 3.12+
* **Управление зависимостями:** Poetry (вместо устаревшего requirements.txt)
* **Тестовый фреймворк:** Pytest
* **Движок браузера:** Playwright (Synchronous API)
* **Управление конфигурацией и данными:** Pydantic v2 + `.env`
* **Логирование:** Loguru (с авто-ротацией лог-файлов)
* **Отчетность:** Allure Framework (+ автоприкрепление скриншотов и логов при падениях)
* **Масштабирование:** pytest-xdist (Параллельный запуск тестов)
* **CI/CD:** GitHub Actions

## 🏗 Ключевые архитектурные паттерны
1. **Page Object Model (POM):** Строгое разделение локаторов от бизнес-логики. Локаторы инкапсулированы через `@property`.
2. **Dependency Injection (Внедрение зависимостей):** Тесты не инициируют классы страниц самостоятельно. Объекты создаются и передаются "под ключ" через фикстуры `conftest.py`.
3. **API Bypass Login:** Использование `storage_state` и внедрения Cookie для обхода UI-авторизации, что сокращает время прохождения тестов.
4. **Arrange-Act-Assert (AAA):** Строгая структура каждого тестового сценария.
5. **Strong Typing (Строгая типизация):** Использование Type Hints везде для обеспечения автодополнения и защиты от ошибок.

## ⚙️ Инструкция по локальному запуску

### 1. Подготовка окружения
Убедитесь, что у вас установлены:
* `Python` версии 3.12 или выше.
* Менеджер пакетов `Poetry` (`pip install poetry`).
* `Allure CLI` (для генерации красивых отчетов).

### 2. Установка зависимостей
Клонируйте репозиторий и выполните команды в корне проекта:
```bash
# Устанавливаем все библиотеки проекта в виртуальное окружение
poetry install

# Скачиваем браузеры для Playwright
poetry run playwright install --with-deps chromium
```

### 3. Настройка переменных окружения
Создайте файл `.env` в корне проекта (он игнорируется Git'ом) и добавьте туда следующие настройки:
```env
BASE_URL=https://www.saucedemo.com/
STANDARD_USER=standard_user
LOCKED_OUT_USER=locked_out_user
PROBLEM_USER=problem_user
VALID_PASSWORD=secret_sauce
```

### 4. Запуск тестов
В проекте настроен автоматический **параллельный запуск** (`-n auto` в `pytest.ini`).
Для запуска всех тестов с максимальной скоростью:
```bash
poetry run pytest
```

Для запуска тестов последовательно (в 1 поток) используйте флаг переопределения:
```bash
poetry run pytest -n 0
```

### 5. Просмотр отчета Allure
После выполнения тестов (даже если они упали) сгенерируйте и откройте отчет командой:
```bash
allure serve allure-results
```
